REACT

REACT

Weather App

I wanted to improve my understanding of React and in doing so, increase my experience in calling to API’s, the result was the “Weather App”. This project allows users to type in the name of their city and recieve a seven day weather report displaying all the usual information you would expect to see including highest/lowest temperture, pressure, humidity and weather icons assocaited with the report.

I used three API’s to complete this project and they are as follows:

  • Googles Geocoding API - Using the city name as a key the API returns the Longitude and Latitutde of that city.
  • OpenWeatherMap API - Having sourced the Longitude and Latitutde, we use that information to search the Open Weather Map API for weather information for the next seven days.
  • RoadGoat API - To give the project a unique touch I wanted a skyline image of the searched city to appear as a back ground image, hopefully tying the project together. Having sourced the cities Longitude and Latitutde using Google’s Geocoding API, I was able to use that information to search RoadGoat and return a image of said city

MERN

MERN

Sound As A Pound

In October of 2022, myself like the rest of the UK and Europe, were concerned with the value of the British Pound against the Dollar. I wanted to create a tool that made it easy to source that information quickly, and what I started out to be a simple Twitter Bot became a full blown MERN project.

Sound As a Pound calls to currencylayer API to get the value of the GBP against the USD. Twice a day at 8am and 5pm GMT that value is then pushed to a Mongo DB database, and also using the Twitter API, that value is tweeted out on @sound_as_apound. The information that was pushed to the Mongo DB database then gets pulled and rendered into a Line Graph using React.

The user then has two options, they can follow @sound_as_apound and get morning and evening updates on the GBP value, or they can visit the website and see a graph illustrating the rise and fall of the GBP since I started plotting the data.

Missing Piece Website

I completed this project as part of my Milestone 3 submission for my Code Institue Diploma. I was tasked with creating a website that displayed the previous skills in my previous projects, but also incorporate Python. The site needed to be response, accessible, interactive and be connected to a database. In order to do this I chose to use Flask as a Python frame work, and connect the site to a MongoDB Database.

This site was created for a company called "Missing Piece", which offers an online platform to connect UK based rental companies together in order to crosshire and pool their resources. The site displays CRUD operations, allowing users to register an account, amend account information, upload hire stock information to a database and amend or delete that information from said database.

Although displaying my Python skills was the main requirement of this project, I still needed to have a key design aesthetic running through out for the site . I used a combination of both Bootstrap and Materalize CSS frame works to help achieve this task.

PYTHON/DJANGO

Glass Jungle Website

I completed this project as a submission for my Milestone 4 project. This final project was the final challenge as part of my Code Institue Diploma. As with the previous projects this project asked me to display all of the previous skills I had learned on the diploma HTML, CSS and Javascript. However, unlike my Missing Piece website that used both a combination of Flask and MongoDB this project required the use of well known python framework Django. The project needed to display CRUD functionality and also allow the user to complete a transaction.

I created an eccommerce site for an online tropical terrarium retailer called Glass Jungle. Glass Jungle offers its clients all of the equipment required to create their own planted terrariums at home. The site displays CRUD operations, allows users to register an account, amend account information, rate and comment on product featured on the sites store and upload posts on the "Share Your Builds!" section of the site. The site also allows vistors to purchase items on the site, they can either register an account to do that or not. I turned to Stripe as it is a popular and well used tool for taking care of payment transactions.

I am extremely proud of this project, as I feel it perfectly displays the skills I have picked up during my diploma. Not only does the site function as I intended, aesthetically I feel it carries a strong indenty that really connects with the Glass Jungles intended target market.

HACKATHON

Hackathons

Having completed the Code Institute Diploma in Software Development I have acquired a strong set of technical skills to use as foundations in which to grow a successful career as a developer. However, what is lacking, is experience in working with a development team.

To combat this potential weakness and increase both my team working and team management skills, I am an avid attendee of the Code Institute Hackathons. Each Hackathon has a main topic and a brief relating to that topic. In May of 2022, I took part in the Earth Day Hackathon. On this occasion the brief was relatively open, the only stipulation being to create an application to help inform the user of the impact of climate change, or to provide a tool on how they could help limit the affect they themselves were having on climate change.

Together with a team of four other developers, in three days we created a website called “The Planeteers. Aimed at young children, the goal was to present useful facts together with a fun game to help educate them on their impact on the climate. Although the tech stack used on this project was relatively simple, the main learning goal was achieved. During the completion of this project, I gained a better understanding of Git, working from branches other than the main one, submitting pull requests and merging pull requests. I also acted as SCRUM master, which allowed me to improve my team management skills. Using Github Projects I was able to ensure communication within the team was constant. This resulted in the team working efficiently and avoiding any stumbling blocks in the form of conflicts.
